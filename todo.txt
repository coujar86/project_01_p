요약:
사용자가 자연어로 질문하면, 
LangGraph가 질의를 분석하고 → 검색 전략을 고르고 → Elasticsearch에서 하이브리드 검색하고 → 결과를 요약/정리해서 답변하는 기능

이 기능은 단순한 1회 호출보다 아래처럼 분기와 상태 관리가 중요:
1.	사용자 질문 분석
2.	질문 유형 분류
	•	단순 키워드 검색
	•	의미 검색
	•	필터 추출 필요
	•	후속 질문
3.	검색 쿼리 생성
4.	Elasticsearch 검색 실행
5.	결과 부족 시 재검색
6.	최종 답변 생성


// 자연어 검색 → 구조화 검색 조건 변환기
예:
“지난달에 올라온, 작성자 kim이 쓴, 고양이 이미지가 있는 글 찾아줘”
출력:
•	기간 필터
•	작성자 필터
•	이미지 태그 필터
•	본문 semantic query



1.	검색 대상 정리
    •	어떤 필드를 AI 검색에 쓸지 정함
    •	예: 제목, 내용, 작성자, 이미지 설명, 작성일
2.	Elasticsearch 검색 방식 정리
	•	키워드 검색
	•	벡터 검색
	•	필터 검색
	•	이 3개를 같이 쓰는 구조로 잡음
3.	벡터화 준비
	•	문서 임베딩 생성
	•	Elasticsearch에 벡터 필드 저장
4.	AI 검색 흐름 설계
	•	사용자 질문 받기
	•	질문에서 조건 추출
	•	검색 실행
	•	결과가 부족하면 질의 수정 후 재검색
	•	최종 결과 요약
5.	LangGraph 노드 만들기
	•	질문 분석 노드
	•	필터 추출 노드
	•	ES 검색 노드
	•	결과 평가 노드
	•	재검색 노드
	•	답변 생성 노드
6.	FastAPI API 연결
	•	/ai-search 같은 엔드포인트 생성
	•	LangGraph 실행 결과를 응답으로 반환
7.	기본 안전장치 넣기
	•	검색 결과 없을 때 처리
	•	LLM 응답 시간 초과 처리
	•	잘못된 필터 추출 방지
8.	간단한 평가
	•	질문 몇 개 정해서
	•	일반 검색보다 더 잘 찾는지 확인
9.	데모/이력서용 정리
	•	문제: 기존 검색은 자연어 질의 처리 한계
	•	해결: LangGraph + ES hybrid search
	•	결과: 검색 정확도/사용성 개선


강조 포인트:
•	자연어 질의를 구조화된 검색 조건으로 변환
•	LangGraph 기반 다단계 검색 오케스트레이션
•	Elasticsearch hybrid retrieval 설계
•	재검색/쿼리 리라이팅 기반 검색 정확도 개선
